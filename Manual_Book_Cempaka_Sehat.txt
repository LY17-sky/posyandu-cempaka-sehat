MANUAL BOOK SISTEM POSYANDU CEMPAKA SEHAT
=========================================

BAB 1: PENDAHULUAN
-----------------------------------------
1.1 Tentang Aplikasi
Sistem Posyandu Cempaka Sehat adalah aplikasi berbasis web yang dirancang untuk membantu pengelolaan data posyandu secara digital. Sistem ini memfasilitasi pencatatan data balita, penimbangan (KMS), jadwal imunisasi, konsultasi kesehatan, serta penyajian laporan yang akurat. Aplikasi ini dirancang dengan antarmuka yang ramah pengguna, memudahkan peran Super Admin, Admin Pos, Bidan, dan Ibu Balita dalam memantau tumbuh kembang anak secara real-time.

1.2 Tujuan Manual
Buku manual ini bertujuan sebagai panduan komprehensif bagi pengguna dalam mengoperasikan Sistem Posyandu Cempaka Sehat. Melalui manual ini, diharapkan pengguna dapat:
- Memahami alur kerja aplikasi secara keseluruhan.
- Menggunakan seluruh fitur yang tersedia secara maksimal dan tepat guna.
- Melakukan administrasi mandiri serta mengatasi kendala-kendala dasar (troubleshooting).

1.3 Prasyarat Sistem
Untuk dapat menggunakan Sistem Posyandu Cempaka Sehat dengan optimal, berikut adalah prasyarat sistem yang dibutuhkan:
- Perangkat: Komputer/Laptop, Tablet, atau Smartphone.
- Sistem Operasi: Windows, macOS, Linux, Android, atau iOS.
- Browser: Google Chrome, Mozilla Firefox, Microsoft Edge, atau Safari versi terbaru.
- Koneksi Internet: Diperlukan koneksi internet yang stabil untuk mengakses sistem berbasis web.

BAB 2: MEMULAI CEPAT
-----------------------------------------
2.1 Login Pertama Kali
Langkah-langkah untuk masuk (login) ke dalam sistem:
1. Buka browser dan akses URL sistem Posyandu Cempaka Sehat.
2. Pada halaman awal, masukkan Username atau Email yang telah terdaftar.
3. Masukkan Password dengan benar.
4. Klik tombol Login.
5. Jika data valid, Anda akan langsung diarahkan ke halaman Dashboard sesuai dengan hak akses (Role) Anda.

2.2 Navigasi Dasar
Aplikasi ini menggunakan sistem navigasi Sidebar (menu di sisi kiri) yang memuat berbagai fitur utama, di antaranya:
- Dashboard: Halaman ringkasan informasi dan grafik.
- Menu Data: Terdiri dari Balita, Timbang, dan Imunisasi.
- Menu Layanan: Meliputi Jadwal Kegiatan dan Konsultasi.
- Laporan & Cetak: Berisi Laporan dan Kartu Identitas/KMS.
- Pengaturan & Admin: Berisi Manajemen Pengguna dan Backup/Restore.

Untuk berpindah antar menu, cukup klik nama menu pada bagian sidebar.

Bagian ini dikhususkan untuk Super Admin dan Admin Pos dalam mengelola struktur inti aplikasi.

> Kelola Role (Akses)
Penjelasan: Super Admin dapat mengatur dan menetapkan akses pengguna (Role) menjadi: Super Admin, Admin Pos, Bidan, atau Ibu Balita saat pendaftaran atau melalui edit akun.

Akses: Super Admin (seluruh akun), Admin Pos (kelola akun Ibu di wilayahnya)

> Backup Database
Penjelasan: Fitur untuk mencadangkan (mengunduh) seluruh data posyandu ke dalam sebuah file (.sql/SQLite). Hal ini penting dilakukan rutin untuk keamanan data.
Akses: Super Admin
Cara Penggunaan: Klik menu 'Backup' -> Tekan tombol 'Mulai Backup' -> Simpan file yang terunduh ke tempat yang aman.
[ KOLOM GAMBAR : Screenshot Menu & Tombol Backup ]

> Restore Database
Penjelasan: Fitur untuk mengunggah dan memulihkan kembali sistem dari file cadangan yang telah diunduh sebelumnya dengan sistem penanganan error (PDOException handling).
Akses: Super Admin
Cara Penggunaan: Klik menu 'Restore' -> Pilih file hasil backup -> Klik 'Proses Restore' -> Tunggu hingga selesai.

BAB 5: TROUBLESHOOTING & FAQ
-----------------------------------------
5.1 Troubleshooting (Pemecahan Masalah)
1. Gagal Login (Username atau Password Salah)
   - Solusi: Pastikan tombol Caps Lock tidak menyala. Jika lupa password, hubungi Admin Pos atau Super Admin untuk bantuan reset password.
2. Grafik KMS Tidak Muncul pada Halaman Detail Balita
   - Solusi: Pastikan data "Timbang" untuk balita tersebut telah diinput (Berat/Tinggi Badan). Grafik memerlukan setidaknya satu data riwayat penimbangan untuk ditampilkan.
3. Peringatan Imunisasi Menunjukkan Status "Terlambat"
   - Solusi: Ini terjadi jika usia balita telah melewati batas jadwal imunisasi. Admin Pos/Bidan perlu mengubah statusnya menjadi "Sudah" melalui menu Imunisasi jika anak tersebut baru saja disuntik.
4. Terjadi Error saat Melakukan Restore Database
   - Solusi: Pastikan file yang diunggah adalah berformat valid (.sql) dan merupakan file yang langsung dihasilkan oleh sistem Cempaka Sehat ini. Jika masalah berlanjut, hubungi teknisi.

5.2 FAQ (Pertanyaan yang Sering Diajukan)
Q: Siapa saja yang dapat mendaftarkan akun di sistem ini?
A: Sistem menyediakan pendaftaran untuk umum bagi "Ibu Balita". Sedangkan untuk role "Admin Pos" dan "Bidan", harus ditetapkan/diberikan izin khusus oleh "Super Admin".

Q: Apakah data balita saya bisa dilihat oleh ibu lain?
A: Tidak. Sistem ini dilengkapi dengan Role-Based Access Control (RBAC). Ibu Balita hanya dapat melihat data anaknya sendiri, dan Admin Pos hanya bisa mengakses data di wilayah Posyandunya saja.

Q: Bagaimana jika koneksi internet terputus saat memasukkan data?
A: Data yang belum berhasil disimpan (sebelum klik tombol Simpan) perlu diisi ulang saat koneksi internet kembali stabil. Sistem ini memerlukan koneksi online ke server.


BAB 6: LAMPIRAN
-----------------------------------------
6.1 Glosarium
- RBAC: Role-Based Access Control, sistem pengaturan izin akses fungsi aplikasi berdasarkan perannya (misal: Super Admin vs Ibu).
- KMS: Kartu Menuju Sehat, sarana mencatat dan memantau pertumbuhan balita.
- Z-Score: Standar deviasi yang ditetapkan WHO untuk mengklasifikasikan status gizi anak (Gizi Buruk, Kurang, Baik, Lebih).
- IDAI: Ikatan Dokter Anak Indonesia, instansi rujukan dalam penentuan jadwal vaksinasi.
